TinyFlow阅读(1)

Jul 11, 2018 1 min read

之后的几篇文章会记录一下tinyflow的阅读笔记。作为本系列的第一篇文章会首先介绍一下tinyflow，然后说一下编译安装的过程中可能会遇到的问题以及解决方法。

TinyFlow简介

TinyFlow是NNVM的实验代码，也可以看做是是tqchen为他的Systems for ML课准备的一个材料。TinyFlow使用2k行代码构建了一个相对完整的深度学习系统，其中：

  • 927行代码与operators相关
  • 734行代码与运行时(execution runtime)相关
  • 71行代码与API相关
  • 233行代码与前端相关

TinyFlow编译

TinyFlow的编译可以参照官方的readme，但是我在实际编译的过程中发现会有一些错误，需要对编译文件进行一些修改。以下记录我的编译过程：

  1. 下载代码 bash git clone https://github.com/tqchen/tinyflow.git ~/tinyflow --recursive

  2. 安装torch

    # in a terminal, run the commands WITHOUT sudo
git clone https://github.com/torch/distro.git ~/torch --recursive
cd ~/torch; bash install-deps;
./install.sh

  3. 在你的~/.bashrc文件最后加入以下几行代码

    export PYTHONPATH=${PYTHONPATH}:/path/to/tinyflow/python:/path/to/tinyflow/nnvm/python
export TORCH_HOME=/path/to/torch/

    source ~/.bashrc

  4. 编译 在tinyflow的根目录下

    make

    在我的环境(Ubuntu 16.04)编译时会提示g++ -std=c++11 -Wall -O2 -msse2 -Wno-unknown-pragmas -funroll-loops -fPIC -Iinclude -Idmlc-core/include -I/mnt/e/ubuntu/tinyflow/nnvm/include -I/mnt/e/ubuntu/torch//install/include -I/mnt/e/ubuntu/torch//install/include/TH -I/mnt/e/ubuntu/torch//install/include/THC/ -shared -o lib/libtinyflow.so build/src/op_tensor.o build/src/op_nn.o build/src/op_special.o build/src/session.o build/src/c_api.o build/src/torch/op_special_torch.o build/src/torch/op_tensor_torch.o build/src/torch/op_nn_torch.o build/src/rtc/op_fusion.o
    -Wl,–whole-archive /mnt/e/ubuntu/tinyflow/nnvm/lib/libnnvm.a -Wl,–no-whole-archive -pthread -lm -L/mnt/e/ubuntu/torch//install/lib -lluajit -lluaT -lTH -lTHC /usr/bin/ld: cannot find -lTHC     的错误，这时需要把Makefile中41行最后的-lTHC删去，然后再重新make就可以完成编译了。

  5. 测试 通过

    python example/mnist_softmax.py

    可以测试看是否编译成功，需要注意的是由于它使用了sklearn的fetch_mldata函数导入mnist数据集，当你本地不存在mnist的时候sklearn会去mldata这个地方下载，但是这个网站已经挂掉很久了，因此你需要自己从这里下载mnist-original.mat文件然后复制到~/scikit_learn_data/mldata目录下。 最终会看到输出[0.9092]表示编译安装成功了。
