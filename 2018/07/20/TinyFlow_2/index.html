TinyFlow阅读(2)

Jul 20, 2018 1 min read

在这篇文章中，主要介绍一下TinyFlow的C代码编译后如何导入python中，从而可以通过python的api调用这些基于C的函数。

在第一篇文章中，我们介绍了如何编译TinyFlow，假设你已经顺利编译了TinyFlow，那么在源代码的lib目录下应该会有一个libtinyflow.so的文件。当我们在python中使用时，一般首先会导入tinyflow

import tinyflow as tf

上述代码会导入以下的代码

"""Tinyflow trial."""
from __future__ import absolute_import as _abs
from . import _base
from nnvm.symbol import *
from . import nn
from . import train

from ._base import *
from ._ops import *

from ._session import Session

from ._util import infer_variable_shapes

其中，在_basy.py的第20行有__builtin__.NNVM_LIBRARY_NAME = "libtinyflow"，也就是将要导入的c库文件的名字。然后在nnvm/_base.py的43行中调用_LIB = _load_lib()使用ctypes.CDLL的方式进行导入。

接下去，需要将使用C语言写的op注册为python可以调用的形式，这一部分的代码在nnvm/symbol.py的338行_init_symbol_module(Symbol, "nnvm")，其具体实现在nnvm/_ctypes/symbol.py中，如下：

def _init_symbol_module(symbol_class, root_namespace):
    """List and add all the atomic symbol functions to current module."""
    _set_symbol_class(symbol_class)
    plist = ctypes.POINTER(ctypes.c_char_p)()
    size = ctypes.c_uint()

    check_call(_LIB.NNListAllOpNames(ctypes.byref(size),
                                     ctypes.byref(plist)))
    op_names = []
    for i in range(size.value):
        op_names.append(py_str(plist[i]))

    module_obj = sys.modules["%s.symbol" % root_namespace]
    module_internal = sys.modules["%s._symbol_internal" % root_namespace]
    for name in op_names:
        hdl = OpHandle()
        check_call(_LIB.NNGetOpHandle(c_str(name), ctypes.byref(hdl)))
        function = _make_atomic_symbol_function(hdl, name)
        if function.__name__.startswith('_'):
            setattr(module_internal, function.__name__, function)
        else:
            setattr(module_obj, function.__name__, function)

将plist定义为C语言中的字符指针，size为一个无符号整型，然后调用_LIB.NNListAllOpNames(ctypes.byref(size), ctypes.byref(plist))获得op的数量和所有op集和的列表。其实现在nnvm/src/c_api/c_api_symbolic.cc中：

int NNListAllOpNames(nn_uint *out_size,
                     const char*** out_array) {
  API_BEGIN();
  NNAPIThreadLocalEntry *ret = NNAPIThreadLocalStore::Get();
  ret->ret_vec_str = dmlc::Registry<Op>::ListAllNames();
  ret->ret_vec_charp.clear();
  for (size_t i = 0; i < ret->ret_vec_str.size(); ++i) {
    ret->ret_vec_charp.push_back(ret->ret_vec_str[i].c_str());
  }
  *out_array = dmlc::BeginPtr(ret->ret_vec_charp);
  *out_size = static_cast<nn_uint>(ret->ret_vec_str.size());
  API_END();
}

然后使用一个for循环将所有op的name保存到python列表op_names中，对于每一个op，首先初始化为OpHandle()，它对应的C type为void *，之后使用_LIB.NNGetOpHandle(c_str(name), ctypes.byref(hdl))就可以得到每个op对应的函数操作的handle，其实现也在nnvm/src/c_api/c_api_symbolic.c中：

int NNGetOpHandle(const char* op_name,
                  OpHandle* op_out) {
  API_BEGIN();
  *op_out = (OpHandle)Op::Get(op_name);  // NOLINT(*)
  API_END();
}

那么，如何根据op_name得到对应的op呢？或者说，每个op对应的名字在哪里注册的呢？ 其实，所有的op都是在src/op_tensor.cc中注册的。以标量相加这个op为例，在op_tensor.cc中，它注册的名字为__add_scalar__：

NNVM_REGISTER_OP(__add_scalar__)
.describe("add symbol with scalar")
.set_num_inputs(1)
.include("ElementwiseOpAttr")
.set_attr<FInplaceOption>("FInplaceOption", InplaceIn0Out0)
.set_attr<FGradient>(
    "FGradient", [](const NodePtr& n,
                    const std::vector<NodeEntry>& ograds){
      return std::vector<NodeEntry>{ograds[0]};
});

通过op的name返回了对应的handle后，使用function = _make_atomic_symbol_function(hdl, name)创建一个原子符号函数(atomic symbol function)，主要是对op进行了一层封装。最后将所有的内部op（如addscalar,subsymbol等）加入名为nnvm._symbol_internal的sys.modules列表中，将其他与网络相关的操作（如conv2d, max_pool，linear等）加入名为nnvm.symbol的sys.modules列表中。

至此完成了将使用C写的op导入到python中的工作，之后，我们就可以如tf.nn.linear(),tf.tanh()等python api构建网络了。
